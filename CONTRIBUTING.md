# Contributing to KavachAI

Thank you for your interest in contributing to KavachAI.

## Project Structure

```
kavachai/
├── specs/                  # Project specifications (requirements, design, knowledge base)
├── docs/                   # Documentation index
├── backend/                # Python FastAPI backend (coming soon)
├── dashboard/              # React SOC dashboard (coming soon)
├── README.md               # Project overview
├── LICENSE                 # MIT License
└── CONTRIBUTING.md         # This file
```

## Getting Started

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Make your changes
4. Commit with clear messages: `git commit -m "Add: description of change"`
5. Push to your fork: `git push origin feature/your-feature`
6. Open a Merge Request

## Commit Message Convention

- `Add:` for new features or files
- `Fix:` for bug fixes
- `Update:` for changes to existing functionality
- `Docs:` for documentation changes
- `Refactor:` for code restructuring

## Code of Conduct

Be respectful, inclusive, and constructive. This project aligns with India's AI Governance Guidelines principle of "People First."
